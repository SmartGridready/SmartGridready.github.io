SmartGridready Declaration

Declaration labels for buildings

As an architect and planner for buildings and residential areas you can label your buildings with the SmartGridready label. A prerequisite to get the label is:

  • Vour building(s) engage an EMS for power management.

  • Several power consumers and producers are controlled by the EMS.

  • The connection to the power-grid is optimized regarding minimization of power-consumption peaks.

  • Energy- and power monitoring is available.

  • (Optional) The EMS allows load restriction and optimization of energy costs by engaging dynamic tariffs.

Declaration labels for Products

As a Product supplier you can declare your Products and get the SmartGridready label.

  1. Define the Functional Profiles supported by your products.

    For a list of currently defined Functional Profiles see : SmartGridready Functional Profile Library

    For more information about Functional Profiles see Functional Profiles

  2. Create an External Interface Definition file that describes the communication interface for your product.

    For detail information about External Interface Definition files see Product Description File

  3. Apply for a SmartGridready declaration.

Declaration labels for Communicator devices

As a Communicator supplier you can declare your Communicator devices and get the SmartGridready label.

There are several possibilities to get your product declared.

  1. Integrate SmartGridready capability into your devices. There are several possibilities to achieve this:

    • You integrate one of the Communication Handler libraries into your product in order to be able to control SmartGridready compliant Products. See SmartGridready Communication Handler Libraries.

    • You use the Intermediary service to communicate with SmartGridready compliant Products.

    • You develop your custom Communication Handler software that is able to load External Interface Definition files to able to communicate with SmartGridready compliant Products.

  2. Implement some functionality that make use of SmartGridready Functional Profiles.

  3. Apply for SmartGridready declaration.
